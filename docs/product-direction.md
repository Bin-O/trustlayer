# プロダクト方向性(2026-07-11 決定)

## 定位
特定技能を知らない総務担当者でも、監査に耐える自社支援ができる「ナビゲーター型」SaaS。

## 設計原則(全機能に優先適用)

### 原則1:同じ情報を二度入力させない
- 一度入力された情報は、書類生成・タスク記録・届出・評価の全場面で自動引用する
- 既存の実践:在留カードAI読取、organization_defaults プリフィル、
  雇用条件変更→随時届出27項目diff自動抽出
- 新機能への制約:
  - 面談フォームは既存情報を全て自動表示し、新規増分のみ入力させる
  - 法定文書はフォーム保存と同時に生成する(二度書き工程を作らない)
  - 紙・画像情報の取込は Vision AI 路線を優先検討する

### 原則2:情報は一箇所に集約、遺漏を構造的に不可能にする
- 三層の集約構造:
  - 個人層:従業員詳細=その人の全て
  - 会社層:ダッシュボード「今やること」=全員の全期限
  - 証拠層:document_generations + support_records=監査対応の底帳
- 新機能への制約:全てのタスク・期限は必ず「今やること」に自動集約される。
  子ページにしか存在しない期限を作ることを禁止。業界パッケージ追加時も
  タスクエンジン経由で自動継承する

## 5つの柱

### 1. ナビゲーター設計
従業員状態駆動のタスクフロー(今やること)が総務のデフォルト画面。
各タスクは三段式(これは何?/どうやる?/記録する)。
10支援業務マトリクスはレポートビュー(監査・投資家向け)に降格。

### 2. 物理業務への価値
ソフトは事を代行しない。「知らない」と「証拠がない」を消す:
- 事前:チェックリスト+LINE提醒
- 事中:多言語テンプレート
- 事後:ワンクリック記録→法定文書自動生成

### 3. 評価は面談の副産物
Phase 2 は独立ページではなく、タスクフロー初の周期タスクとして実装する。
四半期面談タスク→三段式→末尾に雇用主評価→evaluations→
信頼スコア面談時評価15点、で閉環。

### 4. 業界精細化=タスクエンジン(共通)+業界設定パッケージ(データ)
追加タスクチェーン・アラートルール・文案・帳票を業界パッケージとして表現し、
業界専用コードは書かない。運送業の特定活動55号→免許→特技1号の期限
カウントダウン(トラック6ヶ月/バス・タクシー1年)は業界パッケージの
タスクチェーンの一例。

### 5. 専門家連携
デモ前はタスク指引末尾の静的情報リンク(行政書士会等の公的検索ページ)のみ。
マッチングは Phase 3 ビジョン(行政書士法改正の「自社作成支援ソフト」定位を厳守)。
